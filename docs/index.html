// overview

What is HopStat?

HopStat is a network looking glass built for network operators, ISPs, and anyone who needs real-time visibility into routing and reachability from multiple points of presence. It replaces scattered SSH sessions and per-router CLI access with a single, unified web interface and REST API.

Distributed as a single Go binary, HopStat embeds the entire React SPA frontend, SQLite database, schema migrations and static assets — all via Go's embed.FS. Drop the binary on a server, write a 10-line YAML file, and you have a fully functional looking glass with an admin panel. No Node.js, no PostgreSQL, no separate web server required.

HopStat runs in two modes: server mode orchestrates queries across all your nodes — whether they are directly reachable routers (SSH/Telnet) or remote agent instances at geographically distributed POPs. An embedded GoBGP peer provides live BGP table access. Router credentials are encrypted at rest with AES-256-GCM. Query results stream in real-time via Server-Sent Events.

// capabilities

Everything you need.
Nothing you don't.

Full-stack network diagnostics with enterprise-grade security, built for the way network operators actually work.

🔍
BGP Route Lookup
Query live BGP tables for any prefix or subnet. See next-hop, full AS path, origin, local-pref, MED and community strings — sourced from an embedded GoBGP peer connected to your router.
📡
Ping
ICMP echo with configurable packet count. Reports packets sent/received, packet loss percentage, and min/avg/max RTT from any node in your network.
🛣️
Traceroute
Hop-by-hop path tracing with per-hop RTT. Each hop is enriched with GeoIP ASN, organisation name and country — so you see the actual path, not just IP addresses.
📊
MTR (My Traceroute)
Combines traceroute and ping for continuous path analysis. Reports packet loss, sent/received counts, and last/avg/best/worst RTT with standard deviation per hop.
🗺️
AS Path Analysis
Enter any ASN and find every prefix in your BGP table that traverses it. Scans IPv4 and IPv6 address families simultaneously. Results capped for performance.
🌐
Multi-Node
Run any query from any number of nodes simultaneously. Direct SSH/Telnet to routers or via lightweight agent instances deployed at remote POPs — mixed topologies supported.
📍
GeoIP Enrichment
MaxMind ASN + City databases auto-downloaded via your license key and refreshed on a configurable interval. Every traceroute and MTR hop shows organisation, city and country.
🔄
Auto-Update
Built-in self-updater polls GitHub Releases, downloads the new binary, hot-swaps it in place and re-executes the process — zero downtime, no scripts needed.
⚙️
Admin Panel
Full web UI for managing nodes, users, BGP neighbors (IPv4 + IPv6), community string filter rules and audit logs — all protected by JWT authentication.
🔐
Encrypted Credentials
SSH and Telnet credentials stored in SQLite with AES-256-GCM encryption. The encryption key never touches the database — set it via config or environment variable.
📦
Single Binary
React SPA, SQLite migrations, i18n strings and static assets embedded via Go's embed.FS. No Node.js, no separate web server, no external runtime dependencies.
⚡
Real-Time Streaming
Query results stream to the browser via Server-Sent Events. Watch traceroute hops appear one by one as they complete — no polling, no page refresh.

// how it works

Two modes.
One binary.

Deploy as a central server with direct router connections, scatter agents across remote POPs, or combine both. The same binary switches modes with a single flag.

🖥️
Server Mode
Central query orchestrator & web UI

Runs the HTTP API, the embedded React SPA, query engine and BGP session manager. It connects directly to routers via SSH or Telnet, or delegates to remote agent instances. Handles rate-limiting, authentication, audit logging and GeoIP enrichment centrally.

  • HTTP API + React SPA served from :8080
  • Embedded GoBGP peer — live BGP table, no router config changes
  • Rate limiting & brute-force protection per source IP
  • JWT-authenticated admin panel
  • Async audit logging with configurable retention
  • Community string filter rules with severity badges
  • GeoIP enrichment (ASN + city) for every hop
  • TLS via certificate file or auto Let's Encrypt (autocert)
  • Optional: IPv4 and IPv6 BGP neighbor sessions per node
📡
Agent Mode
Lightweight remote POP node

A minimal REST server deployed at remote points of presence. Exposes the /agent/v1/* API consumed by the central server. No database, no frontend, no BGP manager — just query execution. The server initiates all requests; the agent needs no inbound exposure to end users.

  • Minimal binary footprint — same binary, --mode=agent
  • Bearer-token authenticated REST API on :9090
  • Executes ping, traceroute, MTR locally on the POP
  • SSH/Telnet to locally reachable routers
  • No direct user exposure — server proxies all queries
  • Deploy via systemd unit, Docker, or bare metal
  • Works behind NAT — server calls agent, not the reverse
  • Any number of agents can be registered per server instance
// quick start

Running in under 60 seconds.

Download the binary and run it. Config, secrets, and the first admin account are generated automatically on first start.

Linux (x86-64)

bash 
# Download the binary
curl -Lo hopstat \
  https://github.com/HopStat/HopStat/releases/latest/download/hopstat-linux-amd64
chmod +x hopstat
./hopstat --mode=server

# config.yaml is created automatically with cryptographically
# random jwt_secret and credential_key — no openssl needed.
# Admin e-mail and password are printed to stdout on first start:
#
#   ╔══════════════════════════════════════════════════════╗
#   ║          HOPSTAT — FIRST RUN CREDENTIALS             ║
#   ╠══════════════════════════════════════════════════════╣
#   ║  URL      http://localhost:8080/admin                ║
#   ║  Email    admin@lookingglass.local                   ║
#   ║  Password xK9mPqR2vTnLwBcY8jAsDf                    ║
#   ╚══════════════════════════════════════════════════════╝
#
# Change your password in Admin → Users after first login.

ARM64 server? Use hopstat-linux-arm64 instead.  Running an agent at a remote POP? Pass --mode=agent.

Install as a systemd service

bash 
sudo ./hopstat --install-service

# Copies binary  →  /usr/local/bin/hopstat
# Generates      →  /etc/hopstat/config.yaml  (random secrets)
# Writes & enables /etc/systemd/system/hopstat.service
# Starts the service immediately
#
# First-run credentials appear in the journal:
journalctl -u hopstat | grep -A 10 HOPSTAT

The service runs with CAP_NET_RAW + CAP_NET_ADMIN capabilities for ping / traceroute / mtr — no need to run as root.

Docker

docker 
docker run -d \
  -p 8080:8080 \
  -v ./data:/app/data \
  -e LG_ADMIN_PASSWORD=changeme \
  ghcr.io/hopstat/hopstat:latest

# jwt_secret and credential_key are auto-generated if not set.
# Pin them via -e LG_JWT_SECRET=... -e LG_CREDENTIAL_KEY=...
# to survive container restarts.

Customising config.yaml

config.yaml 
server:
  host:      "0.0.0.0"
  port:      8080
  org_name:  "My Network"  # shown in the UI header
  as_number: "AS65000"

# security.jwt_secret and security.credential_key are generated
# automatically on first run — you can leave them as-is or pin
# your own 64-char hex values here.

geoip:
  license_key:     ""   # MaxMind account (free tier works)
  account_id:      ""
  db_dir:          "./data/geoip"
  update_interval: "72h"

update:
  enabled: true

▸ All config fields can be overridden with LG_ prefixed environment variables.
▸ See config.example.yaml for the full reference with all options documented.
